## 📝 需要手动完成的事项

### 1. 创建 `.env.example` 文件

由于系统限制，`.env.example` 文件需要手动创建。请在项目根目录创建该文件，内容如下：

```env
# 智学伴 AI个性化学习平台 - 环境变量配置示例
# 复制此文件为 .env 并填入实际配置值

# 数据库配置
DATABASE_URL=sqlite:///./zhixueban.db

# JWT 配置（生产环境必须修改）
SECRET_KEY=your-secret-key-change-in-production-please-use-strong-random-key
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=43200

# 加密密钥
ENCRYPTION_KEY=default-encryption-key-change-in-production

# AI 配置
DEFAULT_AI_PROVIDER=deepseek
AI_TIMEOUT=120
AI_MAX_RETRIES=3

# 文件上传配置
UPLOAD_DIR=uploads
MAX_UPLOAD_SIZE=10485760
ALLOWED_EXTENSIONS=.pdf,.docx,.pptx,.txt,.md

# 日志配置
LOG_DIR=logs
LOG_LEVEL=INFO

# CORS 配置
CORS_ORIGINS=*

# 种子数据同步
AUTO_SYNC_SEED_DATA=true
PROMPT_SEED_PATH=backend/seed_data/prompts.json
MODEL_CONFIG_SEED_PATH=backend/seed_data/models.json
```

### 2. 检查敏感信息

在上传前，请检查以下文件是否包含敏感信息：

- ✅ 检查所有代码文件中是否有硬编码的 API 密钥
- ✅ 确认没有 `.env` 文件被提交
- ✅ 确认没有数据库文件（`.db`）被提交
- ✅ 确认没有日志文件包含敏感信息

### 3. 检查 `.gitignore` 是否生效

运行以下命令检查哪些文件会被提交：

```bash
git status
```

如果看到不应该提交的文件（如 `venv/`, `node_modules/`, `*.db` 等），请检查 `.gitignore` 文件。

## 🚀 上传到 GitHub 的步骤

### 方法一：使用 Git 命令行

1. **初始化仓库**（如果还没有）：
   ```bash
   git init
   ```

2. **添加所有文件**：
   ```bash
   git add .
   ```

3. **检查状态**（确认没有敏感文件）：
   ```bash
   git status
   ```

4. **提交**：
   ```bash
   git commit -m "Initial commit: 智学伴 AI个性化学习平台"
   ```

5. **在 GitHub 创建新仓库**，然后：
   ```bash
   git remote add origin <your-github-repo-url>
   git branch -M main
   git push -u origin main
   ```

### 方法二：使用 GitHub Desktop

1. 打开 GitHub Desktop
2. 选择 "File" -> "Add Local Repository"
3. 选择项目目录
4. 检查文件变更，确认没有敏感文件
5. 填写提交信息并提交
6. 点击 "Publish repository" 上传到 GitHub

## ⚠️ 重要提醒

### 安全注意事项

1. **永远不要提交**：
   - `.env` 文件（包含真实 API 密钥）
   - 数据库文件（`.db`）
   - 日志文件（可能包含敏感信息）
   - 上传的用户文件

2. **必须修改**：
   - 生产环境中的 `SECRET_KEY`
   - 生产环境中的 `ENCRYPTION_KEY`
   - CORS 配置中的 `CORS_ORIGINS`

3. **建议操作**：
   - 使用 GitHub Secrets 存储 CI/CD 中的敏感信息
   - 在 README 中明确标注需要修改的配置项

### 文件大小限制

- GitHub 单个文件限制：100 MB
- 如果 `Web4.zip` 文件很大，建议排除或删除
- 如果 `documents/` 目录很大，可以考虑排除
